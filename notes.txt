Install K3s on the server
=========================
install docker and add yourself to docker group

do this with myserver=<your dns name for your server>:
curl -sfL https://get.k3s.io | K3S_URL=https://myserver:6443 | sh -

sudo cp  /etc/rancher/k3s/k3s.yaml ~/.kube/mcconfig
sudo chown <your user> ~/.kube/mcconfig

note you do not need to install kubectl on the server but can instead now invoke:
sudo k3s kubectl version

Deploy a dashboard
==================
(see https://rancher.com/docs/k3s/latest/en/installation/kube-dashboard/)

GITHUB_URL=https://github.com/kubernetes/dashboard/releases
VERSION_KUBE_DASHBOARD=$(curl -w '%{url_effective}' -I -L -s -S ${GITHUB_URL}/latest -o /dev/null | sed -e 's|.*/||')
kubectl create -f https://raw.githubusercontent.com/kubernetes/dashboard/${VERSION_KUBE_DASHBOARD}/aio/deploy/recommended.yaml
kubectl create -f dashboard-admin.yaml
kubectl proxy & # Do this on the workstation where you run your browser

get a login token with
kubectl -n kubernetes-dashboard describe secret admin-user-token | grep '^token'

goto
http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy



On a workstation
================
Copy over the kubectl config file
(e.g. for server gknuc @ 192.168.86.32)
scp gknuc:.kube/config ~/.kube/mcconfig
sed -i s/127.0.0.1/192.168.86.32/ ~/.kube/mcconfig
sed -i s/default/minecraft/ ~/.kube/mcconfig

install kubectl https://kubernetes.io/docs/tasks/tools/install-kubectl-linux/#install-using-native-package-management

add your new config file to list of kubectl configs e.g.
echo export KUBECONFIG=~/.kube/mcconfig:~/.kube/config_argus >> ~/.bashrc

kubectl config set-context minecraft --namespace minecraft --cluster minecraft --user minecraft
kubectl config use-context minecraft
to see your old context and switch back:
kubectl config get-contexts

kubectl apply -f minecraft.yaml
